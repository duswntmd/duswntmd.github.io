자바스크랩트 프로그래머가 먼저 코드를 미리 작성해놓은 오픈소스가 npm 이다.
npm은 Node Package Manager의 약자로 웹을 개발할 떄 많은 도움을 받을 수 있다.
node를 사용에 있어 패키지의 버전을 관리하는 파일이 package.json이다.
터미널에 작성하는 코드이다. 콘솔에서 적용할 수 도있다.
npm들어가는건 다 터미널에서 사용하는 것이다.
```
npm init
```
package.json폴더 안에 주요 정보 파일이 생성된다.
```
npm install 설치할 패키지명 ex) express
```
npm의 설치하고자 하면 npm install 다운받을 패키지 이름을 기제하면 관련 패키지를 본인의 사용 파일에서 사용할 수 있다.
설치확인은 프로젝트의 파일을 열서 설치 파일을 보거나 devpendencies에 설치 된것을 볼 수 있다. 
```
npm install --save-dev nodemon
```
이런식으로 설치하면 devDependencies속성 개발용 패키지가 만들어진다.
nodemon은 소스 코드가 바뀔 떄마다 자동으로 노드를 재실행해 주는 패키지이다.
개발중에만 사용되는 패키지로 배포할 경우 개발용 패키지는 배포되지 않는다.
```
"scripts": {
    "start": "nodemon 돌릴 파일 이름.js"
  }
```
위 scripts는 package.json에서 볼 수 있다.
처음 scripsts에는 start: 옆에 node 파일 이름이 있다. 
이것을 nodemon으로 바꾼후 터미널에서 npm 파일 이름을 치면 서버가 돌아간다.
nodemon이 없을시에는 node 파일 이름을 치면 돌아가는 것을 볼 수 있다.

기타 npm 명령어
```
npm outdated
```
업데이트할 수 있는 패키지가 있는지 확인할 수 있다.
```
npm update
```
업데이트 가능한 모든 패키지가 Wandted에 적힌 버전으로 업데이트 된다.
```
npm uninstall 패키지명
```
해당 패키지를 제거하는 명령어이다.
```
npm info 패키지명
```
패키지의 세부 정보를 파악하고자 할 때 사용하는 명령어이다.
```
npm whoami
```
로그인한 사용자가 누구인지 알려준다.
```
npm publish
```
자신이 만든 패키지를 배포할 때 사용한다.
배포를 하기 위해서는 npm 사이트에 회원가입이 되있어야 하고 
이미 누가 올려놓은 파일 이름이 같다면 다른 이름을 사용해 올려야 한다.
```
npm unpublish
```
배포한 패키지를 제거할 떄 사용한다. 24시간 이내에 배포한 패키지만 제거할 수 있다.
이러한 제약이 있는 이유는 의존성 관계 때문인데 다른 사람이 사용하고 있는 패키지를 제거하는 경우를 막기 위해서이다.
